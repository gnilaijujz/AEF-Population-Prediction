# 不透水面 `impervious_mean` · 数据来源说明（GEE 路线）

> 配套脚本：`impervious_export_gee.js`
> 用途：为 New York MSA 导出 **2017 / 2018 / 2019 / 2020 / 2021** 年 tract 级 `impervious_mean`，
> 口径对齐 2020，支撑"空间迁移性 → 时间迁移性"对照研究。

---

## 一、数据来源

| 项 | 内容 |
|----|------|
| 产品 | **Annual NLCD Collection 1 · Fractional Impervious Surface（FctImp，不透水面产品）** |
| 提供方 | MRLC / USGS EROS |
| GEE 资产 | `projects/sat-io/open-datasets/USGS/ANNUAL_NLCD/FRACTIONAL_IMPERVIOUS_SURFACE`（社区目录 sat-io，ImageCollection，逐年一层） |
| 像元值 | 每个 30m 像元被不透水材料（路面/屋顶/混凝土等）覆盖的百分比，**0–100 %** |
| 原生分辨率 | 30m（Albers Equal Area, EPSG:5070） |
| 年份覆盖 | 1985–2025（逐年一层），本研究用 2017–2021 |

> ⚠️ `impervious_mean` **来自不透水产品 FctImp 本身**，**不是**从土地覆盖 LndCov 的类别反算的。
> 时间维度只需这一列 → 只用这一个数据集；2020 的 10 个土地覆盖 `*_pct` 类不做。

---

## 二、处理口径（与 2020 一致）

1. **几何冻结**：用 2020 纽约 tract 矢量（键 `cb_2020__3`），各年只换影像年份、不换边界。
2. **有效值掩膜**：`updateMask(值 ≤ 100)`，剔除填充/nodata（>100），只用 0–100 有效不透水像元。
3. **聚合**：`reduceRegions(reducer = mean, scale = 30)` → 每 tract 不透水比例**均值** = `impervious_mean`（0–100）。
4. **键与输出**：`cb_2020__3` → 输出列 `cb_2020_3`；每年一份 `impervious_newyork_YYYY.csv`（列 `cb_2020_3, impervious_mean`）。
5. **不归一化**：输出原始值；标准化最后统一套 **2020 那把尺子**（勿逐年重标）。

---

## 三、为什么把 2020 也一并重算

- 2020 原是**本地 rasterstats** 算的；GEE 的重投影/取样与本地会有细微差别。
- 让 2017–2021 **五年全走同一 GEE / 同一发布**管线，跨年才逐位可比——
  否则会把"本地 vs GEE""不同发布版本"混成假的年际信号。
- 与本项目 NTL 的处理策略一致（NTL 也选了"GEE + 2020 重算"）。

---

## 四、使用步骤

1. 打开 https://code.earthengine.google.com/ ，粘入 `impervious_export_gee.js`。
2. 改第 22 行 `TRACTS_ASSET` 为你上传的 2020 纽约 tract 资产（与 AEF/NDVI/水体/NTL 同一个）。
3. Run → Tasks 面板出现 5 个任务（2017–2021）→ 逐个点 RUN → 导到 Drive 的 `IMPERVIOUS_export/`。
4. 首次跑先看打印的 tract 数量、第一条属性（确认键 `cb_2020__3`）与不透水集合示例（确认逐年可取）。

---

## 五、验收自查

- [ ] 数值范围 0–100，无负值/无 >100；市中心 tract 接近 100、郊野接近 0。
- [ ] 各年略有变化（城市扩张），非全等（否则可能忘换年份）。
- [ ] 行/键与 2020 tract 一一匹配；缺值 tract 留空不插补。
- [ ] 标准化仍用 2020 尺子。
