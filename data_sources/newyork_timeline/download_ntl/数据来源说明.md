# 夜间灯光 `ntl_sum` · 数据来源说明（GEE 路线）

> 配套脚本：`ntl_export_gee.js`
> 用途：为 New York MSA 导出 **2017 / 2018 / 2019 / 2020 / 2021** 年 tract 级 `ntl_sum`，
> 口径对齐 2020，支撑"空间迁移性 → 时间迁移性"对照研究。

---

## 一、数据来源

| 项 | 内容 |
|----|------|
| 产品 | **VIIRS 夜间灯光年度合成 V2.1（masked 版）** |
| 提供方 | Earth Observation Group (EOG), Colorado School of Mines |
| GEE 资产 | `NOAA/VIIRS/DNB/ANNUAL_V21`（ImageCollection，逐年一层） |
| 波段 | **`average_masked`**（掩膜背景后的年度平均辐亮度） |
| 单位 | 辐亮度 nW·cm⁻²·sr⁻¹ |
| 原生分辨率 | ~500m（15 弧秒≈463.83m） |
| 年份覆盖 | 2012–2021（2022+ 在 `ANNUAL_V22`），本研究用 2017–2021 |

> ⚠️ 必须用 **`average_masked`**（masked 版，和 2020 一致），别用未掩膜的 `average`。

---

## 二、处理口径（与 2020 一致）

1. **几何冻结**：用 2020 纽约 tract 矢量（键 `cb_2020__3`），各年只换影像年份、不换边界。
2. **聚合**：`reduceRegions(reducer = sum, scale = 463.83)` → 每 tract 像元辐亮度**求和** = `ntl_sum`。
   - ⚠️ **求和必须在原生尺度（~500m）上做**；改大改小会让参与求和的像元数变、`ntl_sum` 就不可比。
   - `ntl_sum` 是 2020 口径的"总光量"代理（≈ 平均亮度 × 像元数，未乘像元面积），跨 tract/跨年可比即可。
3. **键与输出**：`cb_2020__3` → 输出列 `cb_2020_3`；每年一份 `ntl_newyork_YYYY.csv`（列 `cb_2020_3, ntl_sum`）。
4. **不归一化**：输出原始值；标准化最后统一套 **2020 那把尺子**（勿逐年重标）。

---

## 三、为什么把 2020 也一并重算

- 2020 原是**本地 rasterstats** 算的；GEE 的重投影/取样与本地会有细微差别。
- 让 2017–2021 **五年全走同一 GEE 管线**，跨年才逐位可比，避免混入"本地 vs GEE"假信号。
- 时间维度最小集只需 `ntl_sum`；2020 的 mean/std/cv/max/min/count/intensity_per_km2 时间维度不做。

---

## 四、使用步骤

1. 打开 https://code.earthengine.google.com/ ，粘入 `ntl_export_gee.js`。
2. 改 `TRACTS_ASSET` 为你上传的 2020 纽约 tract 资产（与 AEF/NDVI/水体/不透水面同一个）。
3. Run → Tasks 面板出现 5 个任务 → 逐个点 RUN → 导到 Drive 的 `NTL_export/`。
4. 首次跑先看打印的 tract 数量、第一条属性（确认键 `cb_2020__3`）。

---

## 五、验收自查

- [ ] 用的是 `average_masked` 波段；数值 ≥0，市区 tract 明显高于郊区。
- [ ] 各年略有变化，非全等（否则可能忘换年份）。
- [ ] 行/键与 2020 tract 一一匹配；缺值 tract 留空不插补；标准化用 2020 尺子。
