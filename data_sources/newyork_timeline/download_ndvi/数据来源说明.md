# 植被 `ndvi_mean` · 数据来源说明（GEE 路线）

> 配套脚本：`ndvi_export_gee.js`
> 用途：为 New York MSA 导出 **2017 / 2018 / 2019 / 2020 / 2021** 年 tract 级 `ndvi_mean`，
> 口径对齐 2020，支撑"空间迁移性 → 时间迁移性"对照研究。

---

## 一、数据来源

| 项 | 内容 |
|----|------|
| 产品 | **Landsat Collection 2 · Tier 1 · L2 年度 NDVI 合成** |
| 提供方 | USGS / Google Earth Engine |
| GEE 资产 | `LANDSAT/COMPOSITES/C02/T1_L2_ANNUAL_NDVI`（ImageCollection，逐年一层） |
| 波段 | `NDVI`（归一化植被指数，范围约 −1~1） |
| 原生分辨率 | 30m |
| 年份覆盖 | 1984–至今，本研究用 2017–2021 |

---

## 二、处理口径（与 2020 一致）

1. **几何冻结**：用 2020 纽约 tract 矢量（键 `cb_2020__3`），各年只换影像年份、不换边界。
2. **年度合成**：`filterDate(YEAR)` → `mosaic()` 拼成该年无缝图。
3. **聚合**：`reduceRegions(reducer = mean, scale = 30)` → 每 tract 的 `ndvi_mean`。
   - ⚠️ **单波段必须显式命名** `Reducer.mean().setOutputs(['ndvi_mean'])`，
     否则会被默认命名成 `mean`，导致该列取不到 → 整列全空。
4. **键与输出**：`cb_2020__3` → 输出列 `cb_2020_3`；每年一份 `ndvi_newyork_YYYY.csv`（列 `cb_2020_3, ndvi_mean`）。
5. **不归一化**：输出原始值；标准化最后统一套 **2020 那把尺子**（勿逐年重标）。

---

## 三、五年同管线

- 脚本 `YEARS=[2017,2018,2019,2020,2021]`，**2020 也一并重算**，让五年全走同一 GEE 管线、跨年逐位可比。
- 与本项目 NTL / 水体 / 不透水面的处理策略一致。

---

## 四、使用步骤

1. 打开 https://code.earthengine.google.com/ ，粘入 `ndvi_export_gee.js`。
2. 改 `tracts` 资产为你上传的 2020 纽约 tract 资产（与 AEF/水体/NTL/不透水面同一个）。
3. Run → Tasks 面板出现 5 个任务 → 逐个点 RUN → 导到 Drive 的 `NDVI_export/`。
4. 首次跑先看打印的 tract 数量、第一条属性（确认键 `cb_2020__3`）。

---

## 五、验收自查

- [ ] `ndvi_mean` 非全空（若全空 → 多半忘了 `setOutputs` 显式命名）。
- [ ] 数值合理：绿地/林地 tract 偏高、密集建成区偏低；范围在 −1~1 内。
- [ ] 各年略有变化，非全等（否则可能忘换年份）。
- [ ] 行/键与 2020 tract 一一匹配；缺值 tract 留空不插补；标准化用 2020 尺子。
