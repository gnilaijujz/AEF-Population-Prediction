# AEF 卫星嵌入（A00–A63）· 数据来源说明（GEE 路线）

> 配套脚本：`aef_export_gee.js`（导出）、`aef_postprocess.py` / `merge_aef_2020.py`（切分/对齐）
> 用途：为 New York MSA 导出 **2017 / 2018 / 2019 / 2021** 年（+已有的 2020）tract 级 64 维 AEF 嵌入均值，
> 口径对齐 2020，作为"空间迁移性 → 时间迁移性"研究的**核心数据**。

---

## 一、数据来源

| 项 | 内容 |
|----|------|
| 产品 | **AlphaEarth Foundations · Satellite Embedding V1（年度）** |
| 提供方 | Google & Google DeepMind（经 GEE） |
| GEE 资产 | `GOOGLE/SATELLITE_EMBEDDING/V1/ANNUAL`（ImageCollection，逐年一层） |

| 年份覆盖 | 2017–2025，本研究用 2017–2021 |

## 二、处理口径（与 2020 一致：★AEF 不做标准化★）

1. **几何冻结**：用 2020 纽约 tract 矢量（键 `cb_2020__3`），各年只换嵌入年份、不换边界。
2. **年度合成**：`filterDate(YEAR)` → `filterBounds(tracts)` → `mosaic()`（64 波段年度嵌入）。
3. **聚合**：`reduceRegions(reducer = mean, scale = 10)` → 每 tract 的 64 维嵌入**均值** A00–A63。
4. **不标准化、不做 log**：AEF 各年直接输出**原始 tract 均值**（嵌入本身量纲固定、天然可比）。
   - 已核实 2020 的 A00 mean≈0.005、|值|<0.3 量级，即原始均值、未误标准化。
5. **键与输出**：`cb_2020__3` → 输出列 `cb_2020_3`；每年一份 `aef_newyork_YYYY.csv`（列 `cb_2020_3, A00..A63`）。
6. **后处理（对齐 2020 交付结构）**：由 `aef_postprocess.py` / `merge_aef_2020.py` 把每年 CSV
   按 **2020 的 b0–b5 tract 名单**切成 6 份 `aef_New_York_b{0..5}_YYYY.csv`，补 `MSA_NAME='New York'`、
   `cb_2020_3 → TRACT_ID` 改名。**沿用 2020 每个 b 文件的 TRACT_ID 集合，不重新分批。** 缺值 tract 留空。

---

## 三、年份说明

- 脚本默认 `YEARS=[2017,2018,2019,2021]`（2020 已有）。
- 如需让 2020 也用同一套 4941/5031 tract 口径重算，把 2020 加进 `YEARS` 即可（多导一份不亏）。
- AEF 侧无"本地 vs GEE"一致性问题——2020 本就是 GEE 导出、各年都输出原始嵌入均值。

---

## 四、使用步骤

1. 打开 https://code.earthengine.google.com/ ，粘入 `aef_export_gee.js`。
2. 改 `tracts` 资产为你上传的 2020 纽约 tract 资产（与 NDVI/水体/NTL/不透水面同一个）。
3. Run → Tasks 面板出现 N 个任务 → 逐个点 RUN → 导到 Drive 的 `AEF_export/`。
4. 首次跑先看打印的 tract 数量、第一条属性（确认键 `cb_2020__3`）。
5. 下载各年 `aef_newyork_YYYY.csv` 后，跑 `aef_postprocess.py` 切成 b0–b5 六份、对齐 2020 表头。

## 五、验收自查

- [ ] 每年 5031 tract；6 个 b 文件的 TRACT_ID 集合与 2020 逐一相等。
- [ ] 列名/顺序/BOM 与 2020 一致（`TRACT_ID, MSA_NAME, A00..A63`）。
- [ ] **数值合理**：A00 mean≈0、|值|<0.3 量级（说明是原始 tract 均值、未误标准化）。
- [ ] 各年略有变化，非全等（否则可能忘换年份）。
- [ ] AEF **不参与** 2020 那把 GIS 标准化尺子（AEF 自身不标准化）。
