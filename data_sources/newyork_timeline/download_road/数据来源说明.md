# 道路 `road_density_local` · 数据获取与处理指南（TIGER 本地路线）

> 用途：为 New York MSA 补齐 **2017 / 2018 / 2019 / 2020 / 2021** 年 tract 级 `road_density_local`，
> 口径对齐 2020，支撑"空间迁移性 → 时间迁移性"对照研究。
>
> ⚠️ 与 AEF/NDVI/水体/NTL/不透水面不同：**TIGER 道路不是 GEE 原生数据集**，
> 本特征走**本地下载 + 矢量叠加**（与 2020 同法），不走 GEE。

---

## 〇、2020 的做法（本方案要对齐的基准）

已核实 2020 口径（来自 `道路特征_数据说明.docx` + 输出 CSV）：

| 项 | 2020 的做法 |
|----|-------------|
| 数据源 | **U.S. Census Bureau TIGER/Line 道路矢量（`layergroup=Roads`）** |
| ⚠️ 尺度 | **county 级** roads（含全部道路）——state 级 `PRISECROADS` 只有 primary+secondary，**取不到 local** |
| 分类 | 按 **TIGER MTFCC** 分：`road_density_local` = **S1400**（local 街道），`road_density_secondary` = S1200 |
| `road_density_local` | tract 内 **S1400 道路总长 ÷ tract 面积**（≈ m/km²） |
| 连接键 | `cb_2020_3`（shapefile 字段 `cb_2020__3`） |
| 是否归一化 | 否，输出原始值（标准化最后统一套 2020 那把尺子） |

> **时间维度最小集只需 `road_density_local`（S1400）** → `road_len_total_m`、`road_density_secondary` 不算。

---

## 一、从哪里下载（2017–2021）

- **数据源**：Census TIGER/Line，按**年份目录**：
  ```
  https://www2.census.gov/geo/tiger/TIGERYYYY/ROADS/
  ```
- **要的文件**：**county 级**道路 zip（含全部道路，含 S1400 local）：
  ```
  tl_YYYY_<5位county FIPS>_roads.zip
  ```
  例：2019 年 Bronx（36005）→ `https://www2.census.gov/geo/tiger/TIGER2019/ROADS/tl_2019_36005_roads.zip`
- **逐年逐县**：对 2017/18/19/20/21 各年、纽约 MSA 涉及的**每个 county** 各下一份，再合并。

**✅ 真实可点开的例子（Bronx 县 36005）——URL 里 `YYYY` 必须换成真实年份，且目录名和文件名两处都要换：**
```
https://www2.census.gov/geo/tiger/TIGER2017/ROADS/tl_2017_36005_roads.zip
https://www2.census.gov/geo/tiger/TIGER2018/ROADS/tl_2018_36005_roads.zip
https://www2.census.gov/geo/tiger/TIGER2019/ROADS/tl_2019_36005_roads.zip
https://www2.census.gov/geo/tiger/TIGER2020/ROADS/tl_2020_36005_roads.zip
https://www2.census.gov/geo/tiger/TIGER2021/ROADS/tl_2021_36005_roads.zip
```
想先看某年全部县的清单，直接点目录页：`https://www2.census.gov/geo/tiger/TIGER2020/ROADS/`（把 2020 换成别的年）。

> TIGER 每年发布一版，2007–至今逐年可得；2017–2021 全覆盖。

### ⚠️ 报 404 / "page has been moved" 的三个常见原因
1. **`YYYY` / `TIGERYYYY` 没替换成真实年份**（最常见）——模板不能照抄，`TIGER2019` 和 `tl_2019_` 两处都要是同一个真实年份。
2. **用了 state 级路径**（`.../PRISECROADS/` 或 `tl_YYYY_us_primaryroads.zip`）——那是 primary/secondary，**没有 S1400 local**。county 级"All Roads"才对：`.../ROADS/tl_YYYY_<FIPS>_roads.zip`。
3. **FIPS 位数错**——必须 5 位（2 位州 + 3 位县，如 `36005`），不足补零。

## 二、纽约 MSA 涉及哪些 county（关键：用你自己的 tract 名单反推，别硬记）

**最稳的办法**：county 名单直接从你**冻结的 2020 tract 名单**里取——`TRACT_ID` 形如
`1400000US36005009200`，`US` 后前 5 位就是 county FIPS（`36005` = NY-Bronx）。取全体 tract 的
唯一 5 位前缀 = 该 MSA 的全部 county，**与你的几何完全自洽**，不受 OMB 口径争议影响。

```python
import pandas as pd
# 用任一份纽约 tract 表（如 aef_newyork_2020.csv 或 gis 的 tract 表）
df = pd.read_csv('aef_newyork_2020.csv', dtype=str)
key = 'TRACT_ID' if 'TRACT_ID' in df.columns else 'cb_2020_3'
fips = sorted({s.split('US')[1][:5] for s in df[key]})
print(len(fips), fips)   # → 这就是要逐年下载的 county FIPS 全集
```

> 参考（NY-Newark-Jersey City, NY-NJ-PA MSA 35620，约 20+ 县，横跨 NY/NJ/PA）：
> NY 36 州的 Bronx/Kings/New York/Queens/Richmond/Nassau/Suffolk/Westchester/Rockland/Putnam/Dutchess/Orange 等，
> NJ 34 州的 Bergen/Hudson/Essex/Union/Passaic/Morris/Middlesex/Monmouth/Ocean/Somerset/Sussex/Hunterdon 等，
> PA 42 州的 Pike。**以上仅供核对，务必以上面脚本从你 tract 名单反推的结果为准。**

## 三、怎么下载（批量）

拿到 FIPS 列表后循环下载 + 解压：

```bash
# FIPS 列表用第二节脚本得到，替换到这里
FIPS_LIST="36005 36047 36061 36081 36085 36059 36103 36119 36087 36079 36027 36071 \
34003 34017 34013 34039 34031 34027 34023 34025 34029 34035 34037 34019 42103"

for Y in 2017 2018 2019 2020 2021; do
  mkdir -p roads_$Y
  for F in $FIPS_LIST; do
    wget -P roads_$Y "https://www2.census.gov/geo/tiger/TIGER$Y/ROADS/tl_${Y}_${F}_roads.zip"
    unzip -o roads_$Y/tl_${Y}_${F}_roads.zip -d roads_$Y
  done
done
```

> 每县 roads zip 通常几 MB；一个 MSA 一年 ~20+ 县，五年合计约上百个文件，几分钟即可下完。

## 四、处理（与 2020 一致）

对每个年份 `YYYY`：

1. **合并该年所有 county 道路**：读入 `roads_YYYY/tl_YYYY_*_roads.shp` 全部拼成一张线图层（geopandas `pd.concat`）。
2. **筛 local 道路**：保留 **`MTFCC == 'S1400'`**（时间维度只要 local）。
3. **投影**：把道路线与 2020 tract 面都投到**等距/等积米制投影**（如 US Albers **EPSG:5070**）后再算长度，保证长度单位是米、口径与 2020 一致。
4. **叠加求长**：道路线与 2020 tract 面**相交（clip/overlay）**，按 tract 汇总相交后的 S1400 线**总长度（m）**。
5. **算密度**：`road_density_local = S1400总长(m) ÷ tract面积`（分母口径同 2020；tract 面积用 `cb_2020_11`(ALAND, m²) 或 `Shape_Area`，与 2020 保持一致 → 单位 ≈ m/km²）。
6. **键与输出**：`cb_2020__3` → 输出列 `cb_2020_3`；每年一份 `road_newyork_YYYY.csv`（列 `cb_2020_3, road_density_local`）。
7. **几何冻结 / 不归一化**：始终用 2020 tract 面，只换道路年份；输出原始值，标准化最后统一套 2020 那把尺子；缺值/无路 tract 记 0 或留空（与 2020 一致）。

> 关键一致点：① 只用 **S1400** 当 local；② 长度在 **5070** 下量、单位米；③ 密度分母与 2020 同一面积口径；
> ④ 用 **2020 tract 几何**，不用各年自带边界。

## 五、验收自查（每年做完必查）

- [ ] **county 级**下载（不是 state 级 PRISECROADS），否则 S1400 全缺、`road_density_local` 全 0。
- [ ] county 名单 = 从 tract 名单反推的全集，无遗漏（漏县 → 该片 tract 路网偏低）。
- [ ] 只筛 `MTFCC=='S1400'`；长度在 EPSG:5070 下计算、单位米。
- [ ] 数值合理：市中心街道密集 → 值高；各年略有变化，非全等（TIGER 逐年更新，差异通常很小）。
- [ ] 行/键与 2020 tract 一一匹配；分母面积口径同 2020；标准化用 2020 尺子。
